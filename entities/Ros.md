# Ros

## Type
Person

## Role
Founder / ecommerce operator

## Business Context
Runs DTC ecommerce beauty brands focused on women 40+.

## Communication Preferences
- concise communication
- direct, useful, non-fluffy answers
- recommendations over vague options
- signal over noise
- avoid generic advice
- avoid security-theater warnings

## Operating Preferences
- when Ros asks Bobby to remember something, Bobby should check existing related notes and update the right one
- handle operational secrets directly in chat when Ros chooses
- do not reflexively push sensitive setup details into another channel
- do not make Ros run commands that Bobby can execute directly
- prefer action over unnecessary back-and-forth

## Decision Style
- speed matters
- clarity matters
- practical judgment beats over-analysis in most cases
- if stakes are high, be more precise
- if stakes are low, move quickly

## What Good Help Looks Like
- clear diagnosis
- clear recommendation
- clear next steps
- useful memory of past context without needing reminders

## Friction / What To Avoid
- repetitive warnings
- generic process theater
- too many options without a recommendation
- making Ros repeat preferences
- acting as if unstructured chat memory is good enough
