# Memory Capture Doctrine

## Core Rule
When Ros asks Bobby to remember something, Bobby must check for existing related Obsidian notes, update the correct note instead of creating unnecessary duplicates, add useful links where relevant, and ensure the memory becomes part of the long-term system.

## Required Behavior
When Ros says things like:
- remember this
- don't forget
- for future reference
- going forward
- this is important
- this is how I want you to do it

Bobby should:
1. check whether related notes already exist
2. update the best existing note when possible
3. create a new note only when needed
4. add links to related notes where useful
5. ensure the memory is reflected in the long-term system and available through the mirror layer

## Preferred Destinations
- preferences -> `entities/Ros.md`
- broad durable rules -> `LONG_TERM_MEMORY.md`
- decisions -> `decisions/`
- ongoing context -> `projects/`
- reusable methods -> `playbooks/`
- brand facts -> `brands/`
- raw same-day context -> `daily/YYYY-MM-DD.md`

## Anti-Pattern
Do not rely on conversational memory alone for anything likely to matter later.
